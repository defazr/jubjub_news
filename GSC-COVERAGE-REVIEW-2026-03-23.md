## 2. /topic/ 94개 미색인 — 부분 동의, 부분 반대

### GPT 권고
의미 없는 태그 삭제, noindex 처리, 콘텐츠 추가

### Claude 의견

**의미 없는 키워드('already', 'best', 'you', 'half', 'worlds', 'latest' 등)**:
- 이건 맞음. 원인은 Claude Haiku 키워드 추출 시 일반 단어 반환 + stopwords 리스트 미포함
- 수정 시점: 지금이 아니라 안정화 후. stopwords 리스트 업데이트는 리스크 낮음

**핵심 토픽(ai, nvidia, bitcoin 등) 미색인**:
- 시간이 해결함. 기사가 쌓이면 콘텐츠 두께 자연 증가
- `topicConcepts.ts`에 이미 30+개 토픽에 대한 SEO 설명문 구현 완료

**noindex 처리 — 반대**:
- topic 페이지를 noindex하면 나중에 다시 색인시키기 더 어려움
- 차라리 sitemap에서 의미 없는 키워드만 제외하는 게 나음

**기사 0개일 때 soft 404 처리**:
- 코드 확인 결과 현재 0개여도 정상 페이지로 렌더링됨
- 나중에 `notFound()` 추가하는 것이 맞음. 지금은 대부분 핵심 토픽에 기사 있으므로 급하지 않음

## 3. /news/ 81개 미색인 — GPT 분석 일부 부정확

### "URL에 원본 매체명 포함" — 부정확

코드 확인 결과 (`src/lib/slug.ts`): **slug는 기사 제목에서만 생성됨.**
원본 매체명이 URL에 포함되는 것이 아니라, 제목에 매체명이 언급된 경우 slug에 반영되는 것.
의도적으로 매체명을 넣은 구조가 아님.

### "단순 요약/스크래핑이라 색인 우선순위 낮음" — 부분 동의

이 사이트는 단순 스크래핑이 아님:
- Claude AI가 한국어+영어 요약 생성
- 키워드 자동 추출 + 관련 기사 연결
- JSON-LD 구조화 데이터 완비
- 81개 미색인은 최근 발행된 기사가 **아직 평가 중**인 것일 가능성 높음 (사이트가 2주밖에 안 됨)

**지금 할 일**: 없음. 기사가 쌓이면서 자연 색인 증가를 지켜봐야 함.

## 핵심 메시지

1. 보고서 분석은 대체로 정확하나, "긴급 수정" 권고는 동의하지 않음
2. 사이트가 2주밖에 안 됨 — 미색인 202개는 아직 Google 평가 중
3. 색인 속도 2→154→308은 매우 양호. 이 추세를 깨뜨리면 안 됨
4. 지금 구조 변경하면 Google 재평가 → Discover 테스트 리셋 위험
5. /news/ slug에 매체명이 포함된다는 분석은 부정확 (제목 기반 생성)
6. 의미 없는 키워드(already, best, you 등)는 향후 stopwords 업데이트로 해결
7. **3~4주 후 색인률 추이를 보고 판단하는 것이 올바른 타이밍**
